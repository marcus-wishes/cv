Research Engineer

06.2016 - * VRVis Developing and extending various full stack applications for managing, analyzing, visualizing and mining big data in neuroscience.

Larvalbrain 2.0 (November 2021 - *)

An extension and continuation of the Larvalbrain project.

I work on converting the frontend from JavaScript to TypeScript and React class components to functional components. We improved upon the ThreeJS 3D volume rendering, as well as on the 3D query spatial query possibilities. The state management was modernized to Redux Toolkit, communication with the backend with React-Query, and we added several components, new Plotly based visualizations, and interactive documentation to the Storybook based in-house component libraries.

In the backend, I am working on performance and data storage improvements of our custom 3D spatial data store in Go, data retrieval using SQLC from a PostgreSQL, and the communication between the different Docker microservices.

BrainTrawler (March 2018 - February 2022)

A visual analytics framework for neuroscientists to iteratively explore heterogeneous big brain data, neural circuit connectivity data, and (single cell) gene expression data.

I developed on the frontend using React with Flow types, and Redux state management. To improve rendering, user interaction, and handling of large datasets, and I created a unique workflow to explore gene expression data as a npm packaged component using Tailwind, TypeScript and Vite. I also further developed the Storybook-based component libraries and Plotly-based visualizations.
In the backend I again created a custom data store in Go to efficiently map 3D coordinates to single cell sample metadata and gene expression data, including vast filtering and querying possibilities on the sample metadata.

Project Larvalbrain (June 2016 - February 2018)

A data management/data mining/visual analytics framework for neuroscientists to explore GAL4 confocal microscopy images of the Drosophila melanogaster larvae.

I worked with JavaScript and React on the frontend, including ThreeJS volume and canvas slice rendering of microscopy images, on the spatial querying methods for the volumetric data, and the communication with the backend. on a custom spatial data store written in Go, and the communication with the Go REST-API backend. I also established the use of Storybook-based component libraries.

My work in the backend included the REST-API and a task scheduler and runner in Go, and different microservices, like a GO/Java image processing service. My main focus of work in the backend was a custom data store to efficiently query and data mine 3D volumetric data in Go.

My work also involved development of a desktop tool tracking the registration progress and preview of confocal microscopy images in an SQLite database using C++11 and Qt5, and qmake.

Diagnostic Storytelling (August 2014 - July 2015)

A project to create automatic reports of diagnostic sessions in hospitals.

In this project I created a flexible web UI with HTML, CSS and JavaScript where the OpenGL volume rendering was done remotely by a prototyping framework named VolumeShop. I extended this framework with a plugin to track the user interaction forwarded from the web UI, to save images of the OpenGL volume rendering, and store visualization settings and user notes in a tree. The tree was rendered on the web UI using D3, and one could jump back to any point stored point during the session. At the end of the session, a presentation was created using an own HTML template engine written in C++11, and a HTML/JavaScript presentation library.
