Work Experience

06.2016 - * VRVis Developing and extending various full stack applications for managing, analyzing, visualizing and mining big data in neuroscience.

Larvalbrain 2.0 (November 2021 - *)

An extension and continuation of the Larvalbrain project.

I worked on an overhaul of the basic architecture, converting the frontend from JavaScript to Typescript and class components to functional component with hooks. We improved upon the 3D volume rendering, as well as on the 3D query spatial query possibilities with heavy performance improvements, and adding more query possibilities. Also the state management got modernized to Redux Toolkit, and we vastly improved the Storybook based in house component libraries, adding several new visualizations using Plotly.

In the backend, I am working on performance and data storage improvements of our custom 3D spatial data store, and the communication between the different micro services.

BrainTrawler (March 2018 - February 2022)

A visual analytics framework for neuroscientists to iteratively explore heterogeneous big brain data, neural circuit connectivity data and (single cell) gene expression data.

I developed on the frontend using React with Flow types and Redux state management to improve rendering, user interaction and handling of large datasets, and I created a unique workflow to explore gene expression data as a plugin component for BrainTrawler, and further developed the Storybook based component libraries.

In the backend I again created a custom data store to efficiently map 3D coordinates to single cell sample meta data and gene expression data, including vast filtering and querying possibilities on the sample meta data.

Project Larvalbrain (June 2016 - February 2018)

A data management/data mining/visual analytics framework for neuroscientists to explore GAL4 confocal microscopy images of the Drosophila melanogaster larvae.

In the web frontend, I worked with JavaScript and React on the volume and slice rendering of the microscopy images, on the spatial querying methods for the volumetric data, and the communication with the backend. I also established the use of Storybook based component libraries.

The backend was written in Go/Java, and included different services, like a REST API, an image processing service, or a volumetric data mining service. My main focus of work in the backend was a custom data store to efficiently query 3D volumetric data.
